Nos fichiers et fonctions

Les différentes fonctionnalités du jeu sont séparées en plusieurs fichiers.

  • main.py
  • accueil.py
  • carte.py
  • joueur.py
  • pioche.py
  • defausse.py
  • partie.py
  • manche.py
  • click.py

Les structures de données utilisées (Carte, Pioche, Defausse, Joueur, Partie) sont sous forme de classe Python.


main.py

Le fichier main.py contient les lignes de code principales.
C'est ici qu'est affiché l'écran d'accueil et où la partie est lancée.

accueil.py

Le fichier accueil.html affiche, comme son nom l'indique, la page d'accueil du jeu.
Le joueur indiquera le nombre de joueurs présents pour la partie, puis sélectionnnera "Cliquer pour commencer".

carte.py

Ce fichier contient la classe Carte, représentant donc une carte du jeu.
Un objet carte a pour attribut :

  • hauteur : Hauteur de la carte, en fonction de la taille de l'écran
  • largeur : Largeur de la carte, en fonction de la taille de l'écran
  • num : Numéro inscrit sur la carte (de -2 à 12)
  • etat: Etat de la carte (ouverte ou cachée)

Il n'y a pas de méthode, seulement une fonction qui permet de recouvrir l'emplacement "carte en main".

joueur.py

joueur.py contient la classe représentant un joueur.
Les attributs sont les suivants :

  • nom : Simplement le numéro du joueur
  • jeu_actuel : Matrice 3x4 representant le jeu du joueur
  • score_individuel : Score du joueur

Ensuite, les méthodes :

  • evol_score() : Met à jour le score du joueur
  • retrait_colonne(defausse, ecran) : Retire une colonne si elle est rempli du même nombre
  • affiche_jeu(ecran) : Affiche le jeu du joueur à l'écran
  • affiche_jeu_vision_ext(ecran) : Affiche le jeu des autres joueurs si la loupe est sélectionnée

pioche.py

Comme son nom l'indique, on retrouve ici la classe de la pioche du jeu.
Un seul attribut :

  • cartes : Liste de cartes (on peut voir cela comme une pile)

Les méthodes :

  • rempli(ecran) : Rempli la pioche avec les 150 cartes du jeu
  • vide() : Vide la liste pour le début d'une manche
  • melange() : Mélange la pioche
  • affiche(ecran) : Affiche la pioche à l'écran (seulement la première carte)
  • est_vide(defausse) : Si la pioche est vide, ajoute les cartes de la défausse à la pioche

defausse.py

Similaire à pioche.py, defausse.py contient la classe représentant la défausse du jeu.
Un seul attribut également :

  • cartes : Liste de cartes

Des méthodes un peu différentes :

  • affiche(ecran) : Affiche la défausse à l'écran (seulement la première carte)
  • ajout_carte(carte) : Ajoute une carte à la défausse
  • retire_carte() : Retire la première carte de la défausse

partie.py

Contient la classe Partie.
Plusieurs attributs :

  • nb_joueurs : Nombre de joueurs dans une partie
  • tab_joueurs : Tableau avec les différents joueurs de la partie
  • pioche : Pioche de la partie
  • defausse : Défausse de la partie
  • carte_en_main : Image affichée à l'emplacement "carte en main"
  • manche_finie : Booléen répresentant l'état de la manche actuelle
  • score : Tableau contenant les scores des joueurs

Les méthodes :

  • fin_partie() : Si un joueur a atteint les 100 pts, renvoie vrai
  • mise_a_jour_score() : Met à jour les scores après une manche
  • ajouter_joueur(joueur) : Ajoute un joueur à l'attribut tab_joueurs

manche.py

C'est dans ce fichier que se déroule une manche entière.
