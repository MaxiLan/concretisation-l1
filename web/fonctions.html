Nos fichiers et fonctions

Les différentes fonctionnalités du jeu sont séparées en plusieurs fichiers.

  • pioche.py
  • defausse.py
  • partie.py
  • carte.py
  • joueur.py
  • robot.py
  • strategie_n1.py
  • strategie_n2.py
  • main.py
  • accueil.py
  • manche.py
  • tour.py


Les fichiers concernant le lancement du jeu:

Main.py

Le fichier main.py contient les lignes de code principales.
C'est ici qu'est affiché l'écran d'accueil et où la partie est lancée.

Accueil.py

Le fichier accueil.html affiche, comme son nom l'indique, la page d'accueil du jeu.
Le joueur indiquera le nombre de joueurs présents pour la partie, puis sélectionnnera "Cliquer pour commencer".

Manche.py

Manche.py est composé de plusieurs fonctions,

  • Ce qui concerne le début d'une manche:
  • lancement_manche
  • retourne_carte
  • joueur_commence: Renvoie l'indice du joueur qui commence
  • Ce qui concerne le déroulement de la manche
  • manche : Réalise une manche entière
  • tour : Réalise le tour d'un joueur
  • Ce qui concerne la fin d'une manche:
  • continuer_partie
  • jeu_fin_manche : Retourne les cartes des jeux après le dernier tour
  • affichage_fin_manche
  • fin_manche : Renvoie vraie si la manche est finie

Tour.py

Tour.py gère toutes les actions d'un tour défini par les clics de l'utilisateur. Elle est donc composé de plusieurs fonctions notamment actions_tour dont découlera l'arboresence de l'utilisation des autres.

  • Concernant l'affichage :
  • affiche_aide
  • cache_aide
  • souris_sur_aide
  • affichage_page_autre_joueur : Affiche la page pour voir les autres jeux
  • voir_autre_jeu : Affiche le jeu d'un autre joueur

Autre fonction :

  • retourne_cartes



Les fichiers concernant les classes:

Partie.py

Contient la classe Partie. Plusieurs attributs :

  • nb_joueurs,
  • tab_joueurs,
  • pioche,
  • defausse,
  • score,
  • manche_finie,
  • carte_en_main

Les méthodes :

  • fin_partie : Renvoie vrai si un joueur a atteint les 100 pts
  • mise_a_jour_score
  • ajouter_joueur

Carte.py

Ce fichier contient la classe Carte, représentant donc une carte du jeu. Un objet carte a pour attribut :

  • hauteur,
  • largeur,
  • num,
  • etat,
  • lieu

Pioche.py

Comme son nom l'indique, on retrouve ici la classe de la pioche du jeu.
Un seul attribut :

  • cartes

Les méthodes :

  • rempli
  • vide
  • melange
  • affiche
  • est_vide

Defausse.py

Similaire à pioche.py, defausse.py contient la classe représentant la défausse du jeu.
Un seul attribut également :

  • cartes

Des méthodes un peu différentes :

  • affiche
  • ajout_carte
  • retire_carte

Joueur.py

Joueur.py contient la classe représentant un joueur. Les attributs sont les suivants :

  • nom,
  • jeu_actuel,
  • score_individuel

Ensuite, les méthodes :

  • debut_manche
  • evol_score
  • retrait_colonne
  • affiche_jeu
  • affiche_petit
  • affiche_jeu_vision_ext

Robot.py

La classe robot hérite de la classe joueur. Elle possède donc les mêmes attributs de base auxquels sont rajoutés les suivants:

  • strategie,
  • ind_carte_cachee,
  • jeu_individuel,

Les méthodes :

  • debut_manche
  • choix_pioche_def
  • choix_placement_carte
  • retourne_hasard

Strategie n1.py

On retrouve ici la stratégie employée par le robot, s'il est en mode difficile.

Le robot ne prend pas les cartes au dessus ou égales à 7 (donc il ne fera pas de colonne plus grande que 7).

Il prend dans la défausse si la carte est négative ou permet d'avancer une colonne en cours, sinon il pioche. Dans ce cas, si la carte est plus grande que 7, il la met dans la défausse, et il retourne une carte au hasard. Sinon, il cherche à compléter une colonne ou remplacer des cartes hautes.

Strategie n2.py

Ce fichier contient une stratégie non finie, basée sur les probabilités d'obtenir certaines cartes en piochant.

Cette stratégie aurait dû être meilleur que la stratégie n°1, mais par faute de temps, elle ne verra pas le jour (pour l'instant).
